First Contact

Most Rust syntax matches C++. Some code that compiles in C++ does not compile in Rust. The compiler errors look opaque at first. Later chapters explain why the compiler rejects these programs and how it detects the problems.

If you've written C++, most Rust syntax will feel familiar. Functions, structs, loops, and vectors work the same way.

fn main() {
    println!("Hello, world!");
}

fn main() {
    let x = 5;
    let y = 10;
    println!("{} + {} = {}", x, y, x + y);
}

fn add(a: i32, b: i32) -> i32 {
    a + b
}

fn main() {
    let result = add(3, 4);
    println!("Result: {}", result);
}

fn main() {
    let v = vec![1, 2, 3, 4, 5];
    println!("Vector: {:?}", v);
    println!("Length: {}", v.len());
}

struct Point {
    x: i32,
    y: i32,
}

fn main() {
    let p = Point { x: 10, y: 20 };
    println!("Point at ({}, {})", p.x, p.y);
}

fn main() {
    let mut count = 0;
    while count < 5 {
        println!("Count: {}", count);
        count += 1;
    }
}

These examples do not compile:

fn main() {
    let v = vec![1, 2, 3];
    let w = v;
    println!("{:?}", v);
}

error[E0382]: borrow of moved value: `v`

fn main() {
    let mut v = vec![1, 2, 3];
    let r = &v[0];
    v.push(4);
    println!("{}", r);
}

error[E0502]: cannot borrow `v` as mutable because it is also borrowed as immutable

fn main() {
    let r;
    {
        let x = 5;
        r = &x;
    }
    println!("{}", r);
}

error[E0597]: `x` does not live long enough

fn main() {
    let mut s = String::from("hello");
    let r1 = &mut s;
    let r2 = &mut s;
    println!("{}, {}", r1, r2);
}

error[E0499]: cannot borrow `s` as mutable more than once at a time

fn get_string() -> &String {
    let s = String::from("hello");
    &s
}

fn main() {
    let s = get_string();
}

error[E0106]: missing lifetime specifier

fn main() {
    let mut v = vec![1, 2, 3];
    for x in &v {
        v.push(*x);
    }
}

error[E0502]: cannot borrow `v` as mutable because it is also borrowed as immutable

fn main() {
    let mut v = vec![1, 2, 3];
    let closure = || v.push(4);
    println!("{:?}", v);
    closure();
}

error[E0502]: cannot borrow `v` as immutable because it is also borrowed as mutable

use std::thread;

fn main() {
    let v = vec![1, 2, 3];
    thread::spawn(|| {
        println!("{:?}", v);
    });
}

error[E0373]: closure may outlive the current function

use std::rc::Rc;
use std::thread;

fn main() {
    let rc = Rc::new(5);
    thread::spawn(move || {
        println!("{}", rc);
    });
}

error[E0277]: `Rc<i32>` cannot be sent between threads safely

These programs contain memory bugs. Rust's compiler rejects all of them. C++ compiles them.

The dangling reference case:

fn main() {
    let r;
    {
        let x = 5;
        r = &x;
    }
    println!("{}", r);
}

r refers to x. The block ends. x is gone. r points to nothing.

The C++ equivalent compiles and runs. It prints garbage, or crashes, or prints 5 by accident. The C++ standard leaves this behavior undefined.

int main() {
    int* r;
    {
        int x = 5;
        r = &x;
    }
    printf("%d\n", *r);
}
